Need Help? Troubleshooting Guide

Quick workarounds for common issues encountered during testing, especially in minimal container configurations.

Fonts are displaying as empty squares (tofu) in Arch/Ubuntu WSL ▼

WSL Linux instances are minimal containers and do not bundle fonts by default. Run this inside your distro to install TrueType fonts:

# Arch Linux:
sudo pacman -S --needed --noconfirm ttf-dejavu fontconfig

# Ubuntu/Debian:
sudo apt-get install -y fonts-dejavu fontconfig
The application starts but nothing displays on the screen ▼

This is caused by graphics driver/Vulkan translation conflicts in WSLg. You can force software rendering to bypass the GPU interface:

LIBGL_ALWAYS_SOFTWARE=1 unified-media-studio
Audio warnings ("pw.conf can't load config client.conf") ▼

These warnings are standard driver messages from PipeWire inside WSL. They do not affect the processing capabilities and can be safely ignored.
